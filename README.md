# Дашборд мероприятий для Tilda — автообновление раз в сутки

## Как это работает
GitHub Actions (бесплатно, по расписанию) → Playwright открывает сайты
headless-браузером → результат пишется в Google Таблицу → Apps Script
отдаёт таблицу как JSON → блок на Tilda показывает карточки.

## Настройка (один раз)

### 1. Google Таблица
1. Создайте новую Google Таблицу.
2. Создайте в ней лист с названием `Events` (именно так, с большой буквы).
3. Скопируйте ID таблицы из URL: `.../spreadsheets/d/ВОТ_ЭТОТ_ID/edit`.

### 2. Сервисный аккаунт Google (чтобы скрипт писал в таблицу без участия людей)
1. console.cloud.google.com → создать проект.
2. Включить Google Sheets API.
3. IAM → Service Accounts → создать сервисный аккаунт → создать ключ (JSON) → скачать файл.
4. Открыть скачанный JSON, скопировать email вида `xxx@xxx.iam.gserviceaccount.com`.
5. В Google Таблице → "Поделиться" → вставить этот email, дать права "Редактор".

### 3. GitHub-репозиторий
1. Создать новый репозиторий, загрузить туда содержимое этой папки.
2. Settings → Secrets and variables → Actions → New repository secret:
   - `SHEET_ID` — ID таблицы из шага 1.
   - `GOOGLE_SERVICE_ACCOUNT_JSON` — содержимое скачанного JSON-ключа целиком.
3. Всё, workflow `.github/workflows/daily.yml` будет запускаться каждый день
   в 08:00 МСК. Можно запустить вручную: вкладка Actions → Daily events scrape → Run workflow.

### 4. Apps Script (отдаёт JSON для Tilda)
1. В той же Google Таблице: Расширения → Apps Script.
2. Вставить код из `apps-script/Code.gs`.
3. Развернуть → Новое развёртывание → Веб-приложение → Доступ "Все, у кого есть ссылка".
4. Скопировать URL — он вида `https://script.google.com/macros/s/XXXX/exec`.

### 5. Tilda
1. Добавить блок "Другое" → T123 (HTML-код).
2. Вставить содержимое `tilda-embed.html`.
3. В строке `var DATA_URL = '...'` вставить URL из шага 4.

## Если для какого-то сайта эвристика не сработала
В `scraper/scrape.js` функция `genericParse` ищет ссылки, рядом с которыми
встречается похожий на дату текст — подходит для большинства карточных
списков, но не универсальна. Если для конкретного сайта список не
собирается (в логах GitHub Actions будет "найдено 0"), нужно дописать
отдельный парсер под структуру именно этого сайта — пришлите мне
HTML/скриншот страницы списка мероприятий, подправлю прицельно.

## Добавление/удаление источников
Список сайтов — в начале `scraper/scrape.js`, массив `SOURCES`. Просто
добавляете объект `{ name, url, topic }`.
